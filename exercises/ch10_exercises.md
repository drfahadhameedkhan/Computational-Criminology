# Chapter 10 exercises

## Network analysis of co-offending and criminal organisation

- Coding task. Construct a co-offending network from arrest records, real or synthetic, in which an edge means two people were arrested for the same incident. Compute degree and betweenness centrality and report how far the two rankings of individuals agree.
- Coding task. Rebuild the same network under two boundary rules, co-arrest within one year and within three years. Report how the network’s size, density, and the identity of its most central individuals change, and write 200 words on what this sensitivity means for operational use.
- Coding task. Using a synthetic network with simulated victimisation events, compute each person’s network exposure to violence, the share of their immediate ties victimised in a prior window, and test whether exposure predicts subsequent victimisation in your simulation.
- Draft the inclusion criteria you would require before a police force could add a person to a gang database, including the evidence threshold, notification, review, and removal. Justify each element against the failures of the Gangs Violence Matrix.

---

Coding tasks above can be started from the matching chapter script (`code/python/ch10_*.py` or `code/r/ch10_*.R`) and the synthetic data in `data/synthetic/`. Where an exercise asks for real data, the data directory README lists open sources you can substitute.
